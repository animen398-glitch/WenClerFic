# Настройка Vercel для WenClerFic

## Настройки в интерфейсе Vercel:

### 1. Корневой каталог (Root Directory)
- Оставьте: `./` (по умолчанию)

### 2. Настройки сборки и вывода (Build and Output Settings)

#### ✅ Команда сборки (Build Command):
- **Включите переключатель** (включите тоггл)
- Оставьте пустым ИЛИ укажите: `npm install`
- Для Node.js приложений обычно можно оставить пустым

#### ✅ Выходной каталог (Output Directory):
- **Оставьте выключенным** (как есть)
- Для Node.js сервера это не нужно
- Это только для статических сайтов

#### ✅ Установить команду (Install Command):
- **Включите переключатель** (включите тоггл)
- Укажите: `npm install`
- Это обязательно для установки зависимостей

### 3. Переменные среды (Environment Variables)

**Удалите пример** (EXAMPLE_NAME) и добавьте следующие:

1. **GOOGLE_CLIENT_ID**
   - Значение: ваш Google Client ID

2. **GOOGLE_CLIENT_SECRET**
   - Значение: ваш Google Client Secret

3. **GOOGLE_REDIRECT_URI**
   - Значение: `https://ваш-проект.vercel.app/api/auth/google/callback`
   - ⚠️ Замените `ваш-проект` на реальное имя проекта после первого деплоя

4. **FACEBOOK_APP_ID** (если используете Facebook)
   - Значение: ваш Facebook App ID

5. **FACEBOOK_APP_SECRET** (если используете Facebook)
   - Значение: ваш Facebook App Secret

6. **FACEBOOK_REDIRECT_URI** (если используете Facebook)
   - Значение: `https://ваш-проект.vercel.app/api/auth/facebook/callback`

### 4. Framework Preset
- Выберите: **Other** (или оставьте автоматическое определение)

## После настройки:

1. Нажмите **"Развертывать"** (Deploy)
2. Дождитесь завершения деплоя (1-2 минуты)
3. После первого деплоя:
   - Скопируйте URL вашего проекта (например: `https://wenclerfic.vercel.app`)
   - Обновите `GOOGLE_REDIRECT_URI` с правильным URL
   - Передеплойте проект

## Важно!

- Файл `vercel.json` уже настроен правильно - Vercel автоматически определит настройки
- База данных SQLite будет работать на Vercel
- Все данные сохраняются в файле `data/wenclerfic.db`

## Если что-то не работает:

1. Проверьте логи деплоя в разделе "Deployments"
2. Убедитесь, что все переменные окружения добавлены
3. Проверьте, что `package.json` содержит правильные зависимости
